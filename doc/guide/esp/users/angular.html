Embedded Server Pages for Angular

Angular is a client-side web framework designed for single page applications and is well-suited for creating dynamic management applications. ESP support Angular by providing an Angular component for the esp generator and a script library to extend Angular for ESP.

Why Angular?

Why use a client-side framework rather than just serve or generate HTML from the web server? The answer is a better user experience. By using a framework such as Angular, you can create a faster, more responsive, more dynamic user interface. You can reduce or eliminate the typical page reload wait times.

There are may client-side web frameworks to choose from, but Angular offers one of the more powerful framworks that requires very little coding to express powerful interfaces. When ESP is configured to receive requests and serve responses in JSON notation, the result can be fast, responsive web applications with minimal load on the embedded server.

This application was created using the ESP Angular support.

Distributed MVC

When using a client-side, single-page architecture (SPA) framework like Angular, the traditional Model-View-Controller paradigm becomes distributed or spread-out over both the server and the client. Both client and server run MVC stacks, but each side emphasises different parts of the MVC paradigm.

On the server side, the application data and server-side data model is stored. Server controllers receive client requests and fully validate the user and request data. The server stores raw view templates and these are served to the client.

On the client side, a simple data model receives and stores server data. Client side controllers manage the client interaction and manipulate model data for the views. View templates are dynamically manipulated and applied to the brower DOM creating real browser pages.

Client / Server Request Protocol

Between the client and server, the typical exchange of HTML pages, style sheets, scripts and media takes place. However with ESP and Angular, this occurs mostly at the start of the application. Thereafter, JSON model data is provided to the client and requests also in JSON form are sent to the server. While additional HTML, scripts, style sheets and medai may be provided to the client, the vast bulk of the traffic is pure JSON. This can make the application much more efficient and responsive.
